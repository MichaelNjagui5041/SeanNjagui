HTML AND CSS GLOSSARY

Section 2-HTML

Elements
An element is a piece of content and the specific meaning it carries, as well as the code used to mark it up.
Tags
The code that delineates an element
Paragraph
Default text is used when coding a paragraph tag
Self-Closing Elements
These are elements that encloses another element.

Section 4- Skeleton Code

Doctype
The doctype simply lets the browser know which version of HTML the document uses.
Title
The title element should contain a unique title indicating the content or purpose of the page.
Head
The head element exists to hold a variety of content that is not for display on the page itself.
Viewport Meta
This code indicates that the browser should not zoom in on the page and should respect any responsive CSS code. The code should always be exactly the same.

Section 5- Basic HTML Elements

Headings
Headings are used as section titles in an HTML document.There are six levels of headings (h1 - h6) and they can be used hierarchically to create multiple levels sub-sections within the content.
Small
The small represents text that is less important, a side comment, copyright, or legal information.
Strong
strong is an inline element used to add importance to text. By default and most commonly, it makes the text bold.
Unordered Lists (ul)
The ul element represents a list of items, where the order of the items is not important — that is, where changing the order would not materially change the meaning of the list.
